# TESTING

This section describes how each requirement of the application was verified during testing.

### User Selection Dropdown

The application includes a dropdown menu containing five users. To test this requirement, we confirmed that the list is generated using the `getUserIds()` function from `storage.js`. When loaded in the browser, the dropdown correctly displayed User 1 through User 5.

### Displaying Bookmarks for the Selected User

We tested the dropdown by selecting different users and checking that the displayed bookmarks updated accordingly. The `displayBookmarks(userId)` function was triggered whenever a different user was chosen, ensuring the correct bookmarks were shown.

### Handling Users with No Bookmarks

To verify this feature, we selected a user who had no bookmarks stored. The application correctly displayed a message informing the user that no bookmarks were available for that account.

### Reverse Chronological Ordering

Several bookmarks were created for the same user at different times. We confirmed that the newest bookmark always appeared first in the list. This behaviour is implemented through sorting by timestamp within the `displayBookmarks()` function.

### Bookmark Information Display

Each bookmark was checked to ensure it displayed all required information. The following elements were visible:

* Bookmark title
* Bookmark description
* Creation date and time

These details are generated by the `createBookmarkElement()` function.

### Bookmark Title Links

We tested the bookmark titles by clicking on them. Each title correctly opened its associated URL in a separate browser tab.

### Copy URL Functionality

The "Copy URL" button was tested for multiple bookmarks. After clicking the button, the copied content was pasted into a text editor and matched the bookmark URL exactly. This functionality is provided by the `copyToClipboard(url)` function.

### Independent Like Counters with Persistence

To test the like feature, we clicked the ❤️ button several times on different bookmarks. We verified that:

* Each bookmark maintained its own like count.
* The displayed count increased correctly.
* Like counts remained unchanged after refreshing the page.

This functionality is handled through the `toggleLike()` and `setData()` functions.

### Bookmark Creation Form

The form was inspected to confirm the presence of all required inputs:

* URL field
* Title field
* Description field
* Submit button

In addition, form submission was tested using both the mouse and the Enter key.

### Adding Bookmarks to the Correct User

A bookmark was added while a specific user was selected. After switching to another user, we confirmed that the new bookmark did not appear there, demonstrating that bookmarks are stored separately for each user.

### Automatic Refresh After Submission

After creating a bookmark, the bookmark list refreshed immediately and displayed the newly added item without requiring the page to be reloaded.

### Lighthouse Accessibility Testing

Accessibility was evaluated using Lighthouse in Chrome DevTools. Tests were performed with users who had bookmarks and with users who had none. In all cases, the application achieved an Accessibility score of 100.
